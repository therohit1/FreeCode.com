    The OSI Model

  • Established in 1947. (Open System Interconnection)
  • By the International Standards Organization (ISO).
  • Worldwide agreement on international standards.
  • Three-fourths of countries in the world are represented in the ISO.
  • An ISO standard that covers all aspects of network communications is the Open Systems Interconnection (OSI) model.
  • Introduced in late 1970’s.


    • Working

  • Layers 1, 2, 3 are the network support layers.
  • They deal with the physical aspects of moving data from one device to another.
  • Layers 5, 6, 7 are the user support layers.
  • They deal with exchange and use of information among unrelated software systems.
  • Layer 4 provides links to the two subgroups and ensures that what the lower layers have transmitted is in a form that the upper layers can use it.
  • Upper Layer generally comprises of software, where as lower layer comprises of hardware & software.
  • Except physical Layer.

    • How does data exchange takes place

  • The data flow starts from Layer 7 (Application) & goes down to the Layer 1 (Physical).
  • This flow is sequential & descending.
  • At each layer, header gets added to the data unit.
  • When it passes through physical layer it gets converted to EM signal.
  • This is transmitted using physical link to the receiver.
  • Reverse process takes place at destination, i.e. the added header & trailers gets removed from the data unit while passing through each layers.
  • Hence the data is made available in its original form when it reaches to layer 7.

    • Individual working of layers in OSI Model

    Physical Layer

  • Type of transmission media. (defines charac’s of interface between devices & Tx Media).
  • Representation of bits. ( stream of bits 0’s & 1’s must be encoded to Elec/optical Sgnls type of encoding).
  • Data Rate. (Defines the duration of bits)
  • Synch of bits. (Tx & Rx must be sync)
  • Line configuration. (Connected through dedicated link/sharing link)
  • Physical Topology. (Mesh, Star, Ring, Bus)
  • Transmission mode. (SDx, HDx, FDx)

    • Data Link Layer

  • Framing. (Large data is sectioned into frames)
  • Physical Addressing (Header is added = Address of receiver)
  • Flow control. (Prevents overloading of data)
  • Error control. (Detect, Re-Tx, lost/damaged frames, adds Trailer to frames at the end)
  • Access control. (Which device has the control over the link)

    • Network Layer

  • Logical addressing. (The network layer adds a header to the packet)
  • Routing. (Internetworks Routes or switch pkts to destination)

    • Transport Layer

  • Service-point addressing. (service-point address/port address) is added)
  • Segmentation and reassembly. (Msg in segments, each having sequence number, helps in reassembling @ Rx, make & break connection)
  • Flow control. (Flow control of data at the end of layer)
  • Error control. (Msg arrives at the Rx without any errors, Correction is done by Re-Tx)

    • Session Layer

  • Dialog control. (Establishes, maintains, and synchronizes the interaction between communicating systems, also decides the modes HDX,FDX)
  • Synchronization.

    • Presentation Layer (Syntax & Semantics Branch which studies meaning of words)

  • Translation.
  • Encryption.
  • Compression.

    • Application Layer

  • Network virtual terminal (is a S/Wr version of physical terminal, allows remote access, PC communicates with S/Wr terminal, in turn talks to host & vice versa)
  • File transfer, access, and management (Allows access to host from remote to make changes or read, manage, control files from local computer)
